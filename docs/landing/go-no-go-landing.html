Built to be trusted

A decision tool earns its verdict on the method, not the interface.

01

Closed form, no Monte Carlo

Every quantity is conjugate Beta-Binomial, computed exactly. The tool is bit-reproducible and never returns a different verdict on re-run.

02

A dual-criterion rule

A Target Value you hope for and a Lower Reference Value that is the minimum worth pursuing (Lalonde), so a device performance goal is the degenerate case.

03

Robustness that means something

The design is flagged under-powered when power at the target falls below 80%, and a prior-sensitivity panel shows how the answer moves across a vague, a skeptical, and an enthusiastic prior.

04

A pre-specification lock

Hashes the prior, thresholds, and planned n, so an interim run is stamped pre-specified or drifted. It states its own limit plainly: a hash proves integrity, not that it predated the data.

05

Grounded in guidance

The risk-difference dual criterion and simulated operating characteristics align with FDA's 2026 draft Bayesian guidance and the go/no-go literature (Lalonde; BOP2-DC).

06

Plain-English routing

Ask "is it worth running?" or "continue or stop?" and the agent picks the right model, writes the query, checks the assumptions, and reports the caveats a text-to-SQL bot skips.

One agent, many methods

The go/no-go layer is one call in a full data scientist. The same agent picks and fits the model any clinical question needs.

Ask in plain English and it routes to the right method, prepares the data, checks the model's assumptions, corrects for multiple comparisons, and reports the caveats a text-to-SQL tool skips. Every number is computed with statsmodels or scikit-learn, not generated.

It picks the model that fits your data

The agent doesn't hard-code a model. It cross-validates several and keeps the winner — so an upload gets the model that fits it, not a default.

Ask for the best predictor and the agent fits logistic regression, a random forest, and gradient boosting, scores each by a cross-validated composite (ROC-AUC, PR-AUC, and balanced accuracy), and returns a leaderboard with the winner. On the UCI heart-failure data it picks the random forest — the same model Chicco & Jurman (2020) chose when they compared models, with serum creatinine and ejection fraction as the top predictors.

How a question becomes an answer

One plain-English question runs an entire analysis. The UI shows every step, with the cost and latency of the run.

01

Retrieve schema

Pulls the right tables and metric definitions from the semantic catalog.

02

Plan the approach

States a hypothesis and picks the model the question actually needs.

03

Write & fix SQL

Runs read-only SQL, and repairs it when a query errors, returns nothing, or comes back all zeros.

04

Statistical guardrail

Confidence intervals, FDR correction, and checks for confounding and Simpson's paradox.

05

Self-review

Critiques its own answer and the assumptions behind the model it chose.

06

Report

The result, its caveats, and the tables it used — exportable to a Word report.

It keeps its own pipeline honest

A pre-flight health check refuses to run on a broken warehouse, and a live monitoring tab watches the data. When a dbt test fails, the agent repairs the pipeline itself.

DETECT

A dbt test fails

An integrity or freshness test goes red on build — a broken column, a null key, a drifted row count.

DIAGNOSE

Find the root cause

The agent traces the failure back through the dbt lineage to the model and column at fault.

REPAIR

Fix and re-run to green

It proposes and applies the fix, rebuilds, and confirms the test passes — the self-healing loop.

Validated on real public data

The demo runs on synthetic data, which has no ground truth. So the agent's own models are run on real, already-analysed datasets and checked against the published literature, across four of its methods — and its model-selection engine lands on each publication's own model.

How it holds

Each reproduction runs the same agent/modeling.py code the live app uses, and each is enforced in CI (tests/test_validation.py): the odds and hazard ratios keep their established magnitude and direction, the AUC stays inside the published band, and the predictive probability stays within 0.001 of the paper's value. A change that quietly breaks the modeling is caught against real ground truth, not synthetic data.
